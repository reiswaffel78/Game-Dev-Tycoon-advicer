## Implement "Unlocked Topics" filter in Game Planner (React+TS+Vite+Wouter+i18next+shadcn)

### Stack constraints
- Frontend: React + TypeScript + Vite + Tailwind + shadcn/ui + Wouter + TanStack Query + react-i18next
- Backend: Express 5 + Drizzle + Zod + Postgres (Neon). SSR for SEO with react-helmet-async.
- IMPORTANT: This feature must NOT affect SSR/SEO output. It is user-specific and should be client-side persisted.

### Goal
Add a user-selectable filter so the Game Planner can restrict recommendations to TOPICS the user has unlocked in their save.
Default behavior MUST remain identical if filter is off or no topics are selected.

### Requirements (MVP)
1) Data model (planner inputs / save state):
   - Add:
     - useUnlockedTopicsFilter: boolean (default false)
     - unlockedTopicIds: string[] (default [])
   - Backward compatible with any stored state.

2) Persist on client:
   - Store unlockedTopicIds in localStorage key `gdt_unlocked_topic_ids_v1`
   - SSR-safe: no window access during server render; read/write in useEffect or guarded.

3) UI (shadcn/ui):
   - Add a section in the Planner page:
     - Switch: "Filter by unlocked topics"
     - Search input
     - Scrollable checkbox list of all topics
     - Buttons: Select all / Clear
     - Counter: X/Y selected
   - Checkbox stores stable topic id (NOT translated text).
   - All UI text via i18next keys.
   - Topic labels are localized (either via i18next key `topics.<id>` or existing localized dataset).

4) Engine integration:
   - Find engine entry that generates planner recommendations.
   - Pre-filter candidate topic list:
     - if useUnlockedTopicsFilter && unlockedTopicIds.length>0:
         effectiveTopics = ALL_TOPICS.filter(id in unlockedTopicIds)
       else:
         effectiveTopics = ALL_TOPICS
     - if filtering results in empty set: fallback to ALL_TOPICS (and show a small warning in UI).
   - Do NOT change scoring weights/logic besides filtering candidate topics.

5) Regression guard:
   - Add a test or a small script to ensure:
     - With filter OFF, outputs are unchanged (snapshot/hash of top N recs).
   - Prefer Vitest if present; otherwise add a dev script.
