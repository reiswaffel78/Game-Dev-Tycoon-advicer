Game Dev Tycoon Advisor – Handoff Prompt

Du übernimmst ein produktionsreifes Fullstack-Webprojekt: einen Berater/Companion für das Spiel "Game Dev Tycoon" (inkl. Netflix Edition). Die App hilft Spielern, optimale Topic-Genre-Kombinationen, Plattformempfehlungen und Slider-Presets zu finden.

Domain: gamedevtycoonadvisor.com

Tech-Stack:

    Frontend: React + TypeScript, Vite, Wouter (Routing), TanStack React Query, shadcn/ui (Radix UI), Tailwind CSS mit CRT-Retro-Theme (grün/amber)
    Backend: Express.js 5.x, Node.js, Drizzle ORM mit PostgreSQL
    SSR: Produktions-SSR mit react-helmet-async, eigener Entry-Server (client/src/entry-server.tsx)
    i18n: react-i18next, 19 Sprachen (en, de, fr, it, es, ko, ja, zh, hi, tr, pt, ru, cs, nl, ar, el, hu, pl, sv)
    Build: Vite (Frontend) + esbuild (Server), Output in dist/

Projektstruktur:

├── client/src/
│   ├── components/      # shadcn/ui Komponenten
│   ├── pages/           # ~19 Seiten (dashboard, research, staff, timeline, checklist, planner, slider-presets, recommender-Seiten, FAQ, handbuch, app-guide)
│   ├── seo/             # SEO-Komponenten (seo.ts, SeoHead.tsx)
│   ├── locales/         # 19 JSON-Sprachdateien
│   ├── hooks/           # Custom Hooks (inkl. useLocalizedRouter)
│   └── lib/             # Utilities (i18n.ts, translate-data.ts, queryClient.ts)
├── server/
│   ├── routes.ts        # API-Routen mit Zod-Validierung
│   ├── storage.ts       # DB-Zugriffsschicht
│   ├── recommendation-engine.ts  # Empfehlungslogik + Multi-Genre-Berechnung
│   ├── static.ts        # i18n-Routing mit 301-Redirects
│   ├── entry-server.tsx # SSR Entry Point
│   └── seed.ts          # DB-Seeding
├── shared/schema.ts     # Drizzle DB-Schema
└── migrations/          # Drizzle-Migrationen

i18n-Routing-System:

    Englisch = Basispfade: /sliders, /research
    Andere Sprachen = Suffix mit Trailing Slash: /sliders/de/, /sliders/fr/
    Express-Middleware in server/static.ts erzwingt kanonische URLs mit 301-Redirects
    Wouter mit Custom Hook für sprachabhängige Navigation
    hreflang-Alternates, Canonical URLs und JSON-LD pro Seite

Recommendation Engine:

    Gewichtete Scoring-Formel: Topic-Genre 40%, Platform-Genre 25%, Platform-Audience 20%, Unlock-Bonus 10%, Cost-Penalty 5%
    Filter: Platform-Genre Fit ≤ -2 wird ausgeschlossen
    Trust-weighted Merge bei widersprüchlichen Datenquellen

Multi-Genre Feature (zuletzt implementiert):

    Offizielle Formel: (Primary Genre × 2 + Secondary Genre) / 3
    API: GET /api/sliders/multi/:primaryGenreId/:secondaryGenreId
    Frontend: Toggle-Button, zwei Genre-Selektoren, Swap-Button, visuelle 2×/1×-Gewichtung
    Genre-Reihenfolge ist relevant (Action/Adventure ≠ Adventure/Action)

SEO:

    Jede Seite hat einzigartige Longtail-Titel-Keywords in allen 19 Sprachen
    Open Graph, Twitter Cards, JSON-LD Structured Data
    sitemap.xml und robots.txt in client/public/
    SSR liefert ~61K voll gerendertes HTML

Datenmodell (Kernentitäten):

    Topics, Genres, Platforms, Audiences
    Beziehungstabellen: Topic-Genre Fits, Platform-Genre Fits, Platform-Audience Fits (Scores -3 bis +3)
    Development Weights: Slider-Presets pro Genre für 3 Entwicklungsphasen (9 Slider: Engine, Gameplay, Story/Quests, Dialogues, Level Design, AI, World Design, Graphics, Sound)
    Source Tracking mit Trust-Levels

Wichtige Konventionen:

    VITE_-Prefix für Frontend-Env-Vars
    shadcn/ui Imports via @/components/ui/
    TanStack Query v5 (nur Objekt-Form)
    Wouter statt React Router
    Niemals vite.config.ts, drizzle.config.ts oder package.json manuell ändern
    DB-Migrationen nur über Drizzle ORM (npm run db:push)

Laufende Commands:

    Dev: npm run dev (Express + Vite HMR auf Port 5000)
    Build: Vite baut Client nach dist/public, esbuild bündelt Server nach dist/index.cjs
