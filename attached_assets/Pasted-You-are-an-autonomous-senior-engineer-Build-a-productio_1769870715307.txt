You are an autonomous senior engineer. Build a production-ready Next.js webapp that recommends optimal setups for the game "Game Dev Tycoon" (including Netflix edition). The app MUST ingest and normalize data from the public web, store versioned snapshots, and provide explainable recommendations with citations per data point.

TECH STACK
- Next.js (latest stable LTS), App Router, TypeScript
- Tailwind CSS
- Prisma + SQLite for local dev (schema must support easy move to Postgres)
- Zod for validation
- Vitest (or Jest) for unit tests

SECURITY
- Follow current Next.js security guidance; ensure dependencies are patched.
- Do not scrape disallowed sources. Hard-block bild.de.
- Rate limit fetches, cache raw HTML, respect robots.txt where reasonable.

CORE FEATURES
1) Data ingestion pipeline (scripts):
   - scripts/ingest/run.ts: runs full ingestion and produces a snapshot JSON + DB upserts
   - scripts/ingest/sources/*.ts: source adapters for:
     a) forum.greenheartgames.com
     b) gamedevtycoon.fandom.com
     c) gaming.stackexchange.com
     d) steamcommunity.com guides (only if accessible publicly)
   - Save raw HTML to data/raw/<source>/<sha256>.html
   - Extract these fact types:
     - TopicGenreFit (---..+++)
     - PlatformGenreFit (---..+++)
     - PlatformAudienceFit (young/everyone/mature: ---..+++)
     - GenreDevWeights (per dev stage: Engine/Gameplay/Story/Dialogue/LevelDesign/AI/WorldDesign/Graphics/Sound weights)
     - Research nodes and prerequisites (cost RP if found)
     - Platform metadata (license cost, dev cost, release date if found)
   - Normalize names (synonyms) and create canonical IDs.

2) Conflict resolution:
   - Implement trust-weighted merge:
     Priority: Greenheart forum > Fandom > StackExchange > Steam guides
   - If conflicting values:
     - keep both evidences
     - choose highest-trust value as primary
     - reduce confidence score
   - Every fact must store:
     - value, confidence, snapshotId, sourceId, rawEvidencePointer

3) Recommendation engine:
   - Given (topic, optional user state), compute top 3 setups (genre + platform + audience)
   - Score formula:
     S = 0.40*TG + 0.25*PG + 0.20*PA + 0.10*Unlock + 0.05*CostPenalty
     Map ---..+++ to -3..+3
   - Filter rules:
     - if PlatformGenreFit <= -2 => exclude
     - if "Unlocked only" and missing prerequisites => exclude
   - Output must include:
     - numeric breakdown of TG/PG/PA/Unlock/CostPenalty
     - citations: list of sources used for each component

4) Slider presets:
   - Produce stage 1-3 presets (0-100) using GenreDevWeights
   - Apply guardrails:
     - top2 sliders >= 40
     - low-importance sliders <= 20
     - minimum 5 for nonzero if needed
   - Must show explicit numbers + explanation.

5) Planner:
   - User can enter current save state:
     year, month, week, cash, fans, owned platforms, unlocked topics/genres, researched items, team skill summary
   - Planner outputs:
     - next 10 recommended releases (topic/genre/platform/audience/size)
     - next 5 research items (with rationale)

UI PAGES (App Router)
- / (dashboard)
- /recommend/topic (topic picker -> results)
- /recommend/genre
- /recommend/platform
- /planner
- /data/snapshots (list snapshots, view diff summary)
- /sources (show whitelisted sources + last fetch time)

DATA MODEL (Prisma)
- Source, Snapshot, Topic, Genre, Platform, Audience
- Fact tables (or generic Fact with type):
  TopicGenreFit, PlatformGenreFit, PlatformAudienceFit, GenreDevWeights, ResearchNode, EngineFeature
- Evidence: store rawEvidencePointer to data/raw.

ACCEPTANCE CRITERIA
- `pnpm i && pnpm dev` works locally
- `pnpm ingest` performs ingestion and creates a new snapshot
- Recommendations show top 3 with score breakdown and citations
- Slider presets show explicit numeric values per stage
- Planner respects "Unlocked only" toggle and user cash constraints
- Tests cover: scoring, conflict resolution, normalization, slider generation
- Provide a clear README with setup + how to add a new source adapter

DELIVERABLE
- A complete repository with code, scripts, prisma schema, sample snapshot, and README.
